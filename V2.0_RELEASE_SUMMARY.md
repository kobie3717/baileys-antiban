# baileys-antiban v2.0.0 Release Summary

## Overview

Version 2.0.0 introduces the **Session Stability Module** — a comprehensive middleware layer for Baileys socket stability, addressing the #1 reported Baileys issue: "Bad MAC / No Session / Invalid PreKey" errors.

## What's New

### 1. Session Health Monitor

Track decrypt success/failure ratio and detect session degradation before it causes bans.

**Features:**
- Sliding window Bad MAC detection (configurable threshold and time window)
- Auto-triggers `onDegraded` callback when threshold exceeded
- Auto-triggers `onRecovered` callback when health improves
- Real-time health stats accessible via `getStats()`

**Example:**
```typescript
import { SessionHealthMonitor } from 'baileys-antiban';

const monitor = new SessionHealthMonitor({
  badMacThreshold: 3,
  badMacWindowMs: 60_000,
  onDegraded: (stats) => console.error('🔴 SESSION DEGRADED'),
  onRecovered: () => console.log('🟢 RECOVERED'),
});

monitor.recordDecryptFail(true); // Bad MAC
monitor.recordDecryptSuccess();
const stats = monitor.getStats();
```

### 2. Typed Disconnect Reason Classification

Classify Baileys DisconnectReason codes into actionable categories with recovery recommendations.

**Categories:**
- `fatal` — Logged out, restart+QR required (401, 428, 440, 515)
- `recoverable` — Transient errors, safe to reconnect (408, 500, 1000)
- `rate-limited` — Back off before reconnecting (429, 503)
- `unknown` — Unknown codes, reconnect with caution

**Example:**
```typescript
import { classifyDisconnect } from 'baileys-antiban';

const result = classifyDisconnect(429);
console.log(result.category);      // 'rate-limited'
console.log(result.backoffMs);     // 300000 (5 minutes)
console.log(result.shouldReconnect); // true
console.log(result.message);       // 'Rate limited by WhatsApp — cool-off period required'
```

### 3. Socket Wrapper with JID Canonicalization

Transparent Proxy wrapper for Baileys socket that auto-canonicalizes JIDs before `sendMessage()` to reduce LID/PN race conditions.

**Features:**
- Auto-canonicalize JIDs using LidResolver before sending
- Integrated health monitoring
- Exposes health stats via socket properties
- Full passthrough of all other socket methods/properties

**Example:**
```typescript
import { wrapWithSessionStability, LidResolver } from 'baileys-antiban';

const resolver = new LidResolver({ canonical: 'pn' });
const sock = makeWASocket({ ... });

const safeSock = wrapWithSessionStability(sock, {
  canonicalJidNormalization: true,
  healthMonitoring: true,
  lidResolver: resolver,
});

// Use safeSock exactly like normal sock
await safeSock.sendMessage('123456@lid', { text: 'hello' });
// ^ Auto-canonicalized to phone number form if mapping exists
```

### 4. AntiBan Class Integration

Session Stability features integrate seamlessly with the main AntiBan orchestrator.

**Example:**
```typescript
import { AntiBan } from 'baileys-antiban';

const antiban = new AntiBan({
  sessionStability: {
    enabled: true,
    canonicalJidNormalization: true,
    healthMonitoring: true,
    badMacThreshold: 3,
    badMacWindowMs: 60_000,
  },
});

const healthMonitor = antiban.sessionStability;
const stats = antiban.getStats().sessionStability;
```

## Migration Guide

### From v1.x to v2.0

No changes required. v2.0 is a pure feature addition with zero breaking changes.

To enable v2.0 features:

```typescript
// v1.x code (still works)
const antiban = new AntiBan({ warmUp: { warmUpDays: 7 } });

// v2.0 code (opt-in)
const antibanV2 = new AntiBan({
  warmUp: { warmUpDays: 7 },
  sessionStability: {
    enabled: true,  // <-- Add this to enable v2.0 features
  },
});
```

## Technical Details

### Architecture
- Pure middleware layer — no Baileys internals modification
- ES6 Proxy for transparent method interception
- TypeScript strict mode compliant
- Zero new runtime dependencies

### Performance
- Minimal overhead: O(1) JID canonicalization lookup
- LRU eviction for memory-bounded resolver
- Sliding window for efficient Bad MAC tracking

### Compatibility
- Node.js ≥16
- Works with Baileys ≥6.0.0
- Works with @oxidezap/baileyrs ≥0.0.8
